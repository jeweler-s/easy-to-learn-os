# 채은
### ❌ 1) 사용자가 컴퓨터 자원에 접근하기 위해 통하는 운영체제의 구성요소는?
커널
    
### ⚠️ 2) 위 질문의 답이 하는 역할은?
- 프로세스 관리
- 메모리 관리
- 파일 시스템 관리
- 입출력 관리
- 프로세스 간 통신 관리

### ✅ 3) 메모리의 종류 - RAM과 ROM의 차이
RAM은 읽거나 쓸 수 있음 ROM은 읽기만 가능
    
### ⚠️ 4) 부팅이란?
컴퓨터를 켰을 때 운영체제를 메모리에 올리는 과정
    
### ⚠️ 5) 부팅 과정
전원 → ROM에 저장된 BIOS 실행 → 하드웨어 작동 여부 점검 → 메모리에 부트스트랩 코드를 올려 실행 → 부트스트랩 코드가 운영체제 프로세스 실행
    
### ✅ 6) 캐시 쓰기 방식 - 즉시 쓰기와 지연 쓰기의 차이
즉시 쓰기는 캐시에 있는 데이터가 변경될 때 즉시 메모리에 반영.
쓰기 성능이 저하되지만 메모리의 최신 값 유지 가능
지연 쓰기는 변경된 내용을 모아 주기적으로 반영(카피백)
메모리와 데이터 사이의 불일치가 있을 수 있음
    
### ❌ 7) 병렬 처리를 위해 고려해야 할 사항
- 상호 의존성이 없어야 한다.
- 각 단계의 시간을 거의 일정하게 맞춰야 한다.
- 전체 작업 시간을 몇 단계로 나눌지 고려해야 한다.

### ⚠️ 8) 레지스터를 사용하는 이유
작업에 필요한 데이터를 미리 CPU 내부에 보관하여 작업 속도를 향상시키기 위해
    
### ⚠️ 9) 직접 메모리 접근(DMA)을 사용하는 이유
데이터 이동에는 CPU 개입이 필요한데 CPU가 할 일이 너무 많아지면 부하가 생기므로 입출력 관리자에게 작업을 넘겨 입출력 관리자가 직접 메모리에 접근하도록 한다.
    
### ✅ 10) CPU와 직접 메모리 접근이 동시에 메모리에 접근하려 하면 어떻게 되는가?
CPU가 양보한다. 직접 메모리 접근이 순서를 훔쳐간 것이라서 사이클 훔치기라고 부른다.

# 금비
### ⚠️ 1) 컴퓨터 구조 (27pg)
컴퓨터의 구조는 크게 사용자, 소프트웨어, 하드웨어로 나뉘며 각각 아래와 같다.
운영체제는 인터페이스와 커널로 나뉜다.

**소프트웨어**
- 응용 프로그램
- 유틸리티
- 운영체제
    - 인터페이스 : 사용자가 자원에 직접 접근하는 것을 막음과 동시에 사용자 편의 도모
    - 커널 : 운영체제의 핵심 기능을 모아놓은 것

**하드웨어**
- CPU, 메모리, 마우스, 모니터 등등

### ✅ 2) 운영체제의 구성 요소 (47~51pg)
운영체제는 커널과 인터페이스로 구성되어 있다.
- 커널 : 프로세스 관리, 메모리 관리, 저장장치 관리와 같은 운영체제의 핵심 기능을 구현한 프로그램
- 인터페이스 : 사용자와 응용프로그램에 인접하여 커널에 명령을 전달하고 실행 결과를 사용자와 응용 프로그램에 리턴한다.

커널 내부에는 시스템 호출과 드라이버가 존재한다.
- 시스템 호출 : 커널이 자신을 보호하기 위해 만든 인터페이스이며, 커널이 제공하는 시스템 자원을 사용하기 위한 함수다. (API, SDK와 유사) 커널에 진입하기 위해서는 시스템 호출을 무조건 거쳐야 한다.
- 드라이버 : 커널과 하드웨어의 인터페이스이며, 커널/하드웨어 제작자마다 다르다.

### ⚠️ 3) 커널의 주요 역할, 현재 자주 사용되고 있는 커널 구조 (51pg)
커널은 다양한 종류의 프로세스에 CPU를 배분하고 작업할 수 있는 환경을 만들어주며,
크게 아래 역할을 수행한다.
- 프로세스 관리 : 프로세스에 CPU 배분, 작업에 필요한 제반 환경 제공
- 메모리 관리 : 프로세스에 작업 공간 배치, 가상 공간 제공
- 파일 시스템 관리 : 데이터를 저장하고 접근할 수 있는 인터페이스 제공
- 입출력 관리 : 필요한 입력과 출력 서비스 제공
- 프로세스 간 통신 관리 : 공동 작업을 위한 각 프로세스 간 통신 환경 지원

현재 대부분의 운영체제에서는 **계층형 구조 커널**을 사용하고 있다.
이는 단일형 구조 커널이 발전된 형태로, 비슷한 기능을 가진 모듈을 묶어 하나의 계층으로 만들고 계층 간 통신을 통해 운영체제를 구현하는 방식이다.

### ✅ 4) CPU의 기본 구성 (87pg)
- 산술논리 연산장치(ALU) : CPU에서 데이터를 연산하는 장치 (사칙 연산, 논리 연산 수행)
- 제어장치 : CPU에서 작업을 지시하는 장치
- 레지스터 : 작업에 필요한 데이터를 CPU 내부에 보관하는 장치

**사용자 가시 레지스터**
- 데이터 레지스터(DR) : 메모리에서 가져온 데이터를 임시로 보관, 범용 레지스터로 불림
- 주소 레지스터(AR) : 데이터 또는 명령어가 저장된 메모리의 주소

**특수 레지스터**
- 프로그램 카운터(PC) : 다음에 실행할 명령어의 주소 기억
- 명령어 레지스터(IR) : 현재 실행중인 명령어 저장
- 메모리 주소 레지스터(MAR) : 메모리에서 데이터를 가져오거나 반대로 메모리로 데이터를 보낼 때 주소 지정 시 사용
- 메모리 버퍼 레지스터(MBR) : 메모리에서 가져온 데이터나 메모리로 옮겨 갈 데이터 임시 저장, 메모리 주소 레지스터와 항상 함께 동작

### ✅ 5) 두 변수를 선언하여 덧셈 연산을 진행할 때 CPU의 명령어 처리 과정에 대하여 설명하시오 (89pg)
1. 제어 장치가 메모리에 있는 첫 번째 변수의 값을 레지스터로 옮기라는 명령 내림 
2. 제어 장치가 메모리에 있는 두 번째 변수의 값을 레지스터로 옮기라는 명령 내림
3. 제어 장치가 레지스터에 저장된 값을 ALU로 보내 더하라는 명령을 내림, 결과 값은 레지스터에 임시 보관
4. 제어 장치가 레지스터의 값을 메모리로 옮기라는 명령을 내림

⇒ ALU, 제어장치, 레지스터가 협업하여 처리

### ⚠️ 6) 메모리에서 데이터가 레지스터에 적재되는 과정을 설명하시오 (91pg)
1. PC에 실행해야 하는 코드 행 번호가 저장되어 있고 LOAD 명령어가 실행되면 IR에 LOAD 명령어 저장
2. 제어 장치가 IR에 있는 명령어를 해석하여 특정 메모리에 있는 데이터를 가져오라는 제어 신호를 보냄
3. MAR에는 가져와야 하는 데이터가 담긴 메모리 주소가 저장되고, 메모리 관리자가 메모리에 있는 데이터를 CPU로 옮긴다. 메모리 관리자는 MAR에 있는 주소를 읽은 후 해당 주소에 존재하는 값을 MBR로 보낸다. (메모리 관리자가 레지스터에 접근 불가하므로)
4. 제어 장치는 MBR에 저장된 값을 레지스터로 옮긴다.

### ✅ 7) 시스템 버스와 시스템 버스(FSB)의 종류 (83pg, 93pg)
시스템 버스는 메모리와 주변장치를 연결하는 버스이며 메인보드의 동작 속도를 의미한다.

**시스템 버스 종류**
- 제어 버스 : CPU가 메모리와 주변 장치에 제어 신호를 보내기 위해 사용, 양방향
- 주소 버스 : 메모리 주소 레지스터와 연결된 버스, 데이터를 읽거나 쓸 때 메모리나 주변장치에 위치 정보를 보내기 위해 사용, 단방향
- 데이터 버스 : 메모리 버퍼 레지스터와 연결된 버스, 양방향

### ❌ 8) 캐시와 레벨별 캐시의 구조 (106pg)
**캐시의 종류**
- 일반 캐시 : 명령어와 데이터의 구분 없이 모든 자료를 가져오는 캐시
- 특수 캐시 : 명령어와 데이터를 구분하여 가져오는 캐시

**캐시의 레벨**
- L1 캐시 : CPU 레지스터에 직접 연결되는 명령어 캐시와 데이터 캐시를 말함
- L2 캐시 : 메모리와 연결되는 일반 캐시

### ⚠️ 9) 인터럽트 방식의 동작 과정을 인터럽트 벡터와 함께 설명하시오 (109~111pg)
1. CPU가 입출력 관리자에게 입출력 명령을 보낸다.
2. 입출력 관리자는 명령받은 데이터를 메모리에 가져다 놓거나 메모리에 있는 데이터를 저장장치로 옮긴다. 이때 CPU만 접근 가능한 메모리에 입출력 관리자가 접근하기 위해 직접 메모리 접근(DMA) 권한을 가진다.
3. 데이터 전송이 완료되면 입출력 관리자는 CPU에 완료 신호를 보낸다. 이때 장치 이름 대신 장치별 사용하는 고유 번호인 IRQ를 이용한다.

이를 입출력 작업 하나당 한 번씩 진행하면 비효율적이므로 인터럽트 벡터라고 불리는 여러 개의 인터럽트 배열을 사용하여 CPU에게 여러 개의 작업의 인터럽트를 알린다.

### ✅ 10) CPU 코어 수와 멀티 스레딩이 성능을 향상시키는 이유를 설명하시오
CPU 성능을 향상시키기 위해 CPU의 핵심 기능을 가진 코어를 여러 개 만들거나 동시에 실행 가능한 명령의 개수를 늘리는 방법을 사용한다.
멀티 코어 시스템을 통하여 하나의 칩에 CPU 코어를 여러 개 만들어 여러 작업을 동시에 처리할 수 있도록 하며, 하나의 코어에서 2개 이상의 명령어를 처리하는 명령어 병렬 처리를 통해 여러 개의 스레드를 동시에 처리할 수 있도록 한다.
이로 인해 하나의 칩에서 명령어 하나가 아닌 여러 명령을 처리함으로써 처리 성능이 향상된다.
